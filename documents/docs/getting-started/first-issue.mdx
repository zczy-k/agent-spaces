# 创建第一个议题

进入工作空间后，你可以通过创建议题（Issue）让 Agent 开始工作。

## 什么是议题？

议题是你向 Agent 提出的开发需求。它可以是：

- 一个新功能的需求描述
- 一个 Bug 的修复请求
- 一段代码的重构建议
- 任何需要 Agent 完成的开发任务

## 配置 Agent 预设

在创建议题之前，确保已配置至少一个 Agent 预设。

1. 进入工作空间的「Agent 配置」面板
2. 点击「添加 Agent」
3. 填写配置：
   - **名称**：给 Agent 起个名字，如 "Claude 助手"
   - **角色**：选择 agent — 这是实际写代码的角色
   - **运行时**：选择 Claude Code 或 Codex
   - **模型**：选择要使用的 AI 模型
   - **API Key**：填入你的 API Key（如果未在环境变量中配置）
4. 保存配置

## 创建 Workflow 模板

在创建议题之前，建议先创建一个 Workflow 模板来定义 Agent 的执行流程。

1. 在左侧导航点击「Workflows」
2. 点击「创建 Workflow」
3. 在 DAG 编辑器中拖入 Agent 节点，连线定义执行依赖
4. 为每个节点绑定对应的 Agent 预设
5. 保存模板

详见 [Workflow 编辑器](/docs/features/workflow)。

## 创建议题

1. 在左侧面板点击「议题」标签
2. 点击「新建议题」按钮
3. 填写议题信息：
   - **标题**：简洁描述需求，如 "添加用户登录功能"
   - **描述**：详细说明需求内容和期望结果
   - **Workflow 模板**：选择要使用的 Workflow 模板
4. 点击创建

## 自动化执行流程

议题创建后，系统会根据是否绑定了 Workflow 模板决定执行方式：

### 有 Workflow 模板

1. 系统加载 Workflow 模板
2. 将 Workflow 中的节点映射为可执行的 Task
3. 按依赖关系调度执行（无依赖的 Task 并行执行）
4. 所有 Task 完成后，Issue 标记为 completed

### 无 Workflow 模板

Issue 将进入 error 状态，需要手动绑定 Workflow 或通过频道聊天 @mention Agent 执行。

## 查看执行过程

Agent 执行任务时，你可以实时看到：

- 工具调用链（读取文件、搜索代码、编辑文件等）
- 中间输出和思考过程
- Token 使用量统计
- 工具详情（输入参数、输出结果、代码 diff）

你也可以在频道聊天中直接 @mention Agent 触发执行，不走 Workflow 编排流程。
