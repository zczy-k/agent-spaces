# 议题管理

议题（Issue）是你与 Agent 协作的核心入口。通过创建议题并绑定 Workflow 模板，你可以将开发需求交给 Agent 自动处理。

## 议题生命周期

一个议题会经历以下状态：

| 状态 | 说明 |
|------|------|
| 待处理 (Open) | 刚创建，等待启动自动化 |
| 进行中 (In Progress) | Agent 正在通过 Workflow 执行任务 |
| 已完成 (Done) | 所有任务执行完毕 |
| 已关闭 (Closed) | 手动关闭或标记为不需要 |
| 错误 (Error) | 未绑定 Workflow 或执行过程中出错 |

## 创建议题

1. 在工作空间中打开议题面板
2. 点击「新建议题」
3. 填写标题和描述
4. 选择 Workflow 模板（必选，否则 Issue 无法自动执行）

**写好描述的技巧：**
- 明确说明需求目标和期望结果
- 提供必要的上下文信息
- 如果涉及特定文件或模块，直接说明
- 如果有参考代码或设计稿，附上链接

## Workflow 驱动的任务分解

议题启动自动化后，系统会：

1. 加载选定的 Workflow 模板
2. 将 Workflow 中的节点映射为可执行的 Task
3. 按 DAG 拓扑排序确定执行顺序
4. 无依赖的 Task 并行执行
5. 所有 Task 完成后 Issue 标记为 completed

每个 Task 包含：
- 绑定的 Agent 预设
- 执行状态
- 工具调用记录和代码变更

详见 [Workflow 编辑器](/docs/features/workflow)。

## 议题评论

在议题详情页可以添加评论，支持富文本格式。评论用于：

- 补充需求信息
- 与 Agent 交互讨论
- 记录决策和变更

## 议题重试

如果 Issue 执行失败进入 error 状态，可以通过重试机制重新启动自动化流程。

## 替代方式：频道聊天

除了通过 Issue + Workflow 自动编排外，你也可以在频道聊天中 @mention Agent 直接触发执行，适合快速下达指令。
