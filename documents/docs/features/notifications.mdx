# 通知中心

通知中心将 Agent 的状态变更推送到你常用的通讯工具或设备，让你随时随地掌握项目进展。支持外部通知渠道和应用内通知两种方式。

## 外部通知渠道

### 飞书（Lark）

通过飞书 Bot 推送通知：

1. 在飞书开放平台创建应用，获取 App ID 和 App Secret
2. 在项目设置中配置飞书通知
3. 扫码授权后，Agent 状态变更会自动推送到飞书群

### 企业微信（WeChat Work）

通过企业微信 Bot 推送通知：

1. 在企业微信管理后台创建应用
2. 在项目设置中配置企业微信通知
3. 扫码登录后，通知自动推送到企业微信

### Native 通知

支持 Tauri 桌面客户端和浏览器的原生通知：

- **Tauri** — 使用系统原生通知 API，支持 macOS、Windows、Linux
- **Browser** — 使用 Web Notification API，需用户授权

Native 通知无需额外配置，开箱即用。

## 应用内通知

Agent Spaces 内置应用内通知系统，在工作空间内直接接收通知：

### 通知类型

| 通知类型 | 触发场景 |
|----------|----------|
| `issue_completed` | Issue 执行完成 |
| `issue_failed` | Issue 执行失败 |
| `task_completed` | Task 执行完成 |
| `task_failed` | Task 执行失败 |

### 通知操作

- 在侧边栏通知图标查看未读通知数量
- 点击打开通知中心对话框，查看所有通知
- 标记单条通知为已读
- 一键标记全部已读
- 清空所有通知

### WebSocket 推送

通知通过 WebSocket 实时推送：

- `notification.created` — 新通知创建
- `notification.cleared` — 通知被清空

## 通知内容

以下事件会触发通知：

- 议题状态变更（创建、进行中、完成、关闭、失败）
- 任务执行进度更新（完成、失败）
- Agent 执行完成或失败
- 代码审核结果

## Bot 斜杠命令

在飞书或企业微信中与 Bot 对话时，可以使用内置命令远程操控 Agent：

| 命令 | 说明 |
|------|------|
| `/status` | 查看当前议题和任务状态 |
| `/list` | 列出所有议题 |
| `/agents` | 查看已配置的 Agent 预设列表 |
| `/workspace` | 查看工作空间信息 |
| `/usage` | 查看 Token 用量统计 |
| `/help` | 查看所有可用命令 |

Bot Agent 支持自然语言对话和 16 个内置斜杠命令。

## 配置方式

通知配置在工作空间的「项目设置」面板中：

1. 进入工作空间
2. 打开项目设置
3. 切换到「通知配置」标签
4. 选择通知渠道并填写凭证
5. 配置需要通知的事件类型
6. 绑定 Bot Agent（用于处理外部消息）
7. 保存配置

系统支持后端重启后自动恢复通知服务（通过 `serviceRunning` 配置项）。

## Robot Account 凭证管理

Robot Account 是集中管理飞书/企微通知凭证的机制，避免在每个工作空间中重复配置：

### 凭证管理

- 在项目设置的「Robot Accounts」标签中管理凭证
- 支持创建、编辑、删除 Robot Account
- 每个 Robot Account 存储飞书或企微的应用凭证
- 工作空间通过 `robotAccountId` 引用凭证，不直接存储敏感信息

### 企业微信 QR Code 登录

支持全局企业微信 QR Code 扫码登录，系统自动创建 Robot Account 凭证：

1. 在 Robot Accounts 标签点击「企微扫码登录」
2. 扫描二维码完成授权
3. 系统自动创建包含凭证的 Robot Account
4. 工作空间可直接引用该凭证

### 优势

- **凭证隔离** — 敏感凭证集中存储，工作空间只引用 ID
- **复用** — 多个工作空间可共享同一个 Robot Account
- **安全** — 修改凭证只需更新一处，所有引用自动生效
